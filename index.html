About Edis Project

Edis is an advanced communication protocol engineered for maximum security within high-stakes local area networks (LAN). The project focuses on mitigating internal threats and preventing data breaches in critical infrastructures such as power plants and medical facilities.

By leveraging a Zero-Trust security model and hardware-backed isolation (TPM integration), Edis ensures that cryptographic keys remain protected even if a client terminal is physically compromised. Security isn't just a feature in Edis; it is the core of the system's architecture.

Our mission is to bridge the gap between rigorous, military-grade security and a seamless user experience (UX), allowing personnel to exchange sensitive data without navigating the complexities of manual encryption management.

Security Architecture Pillars

TPM

Hardware Isolation

TPM 2.0 Integration

Keys are never exposed to the OS. All cryptographic operations take place within the Trusted Platform Module.

E2E

Zero-Knowledge

Asymmetric Hybrid Encryption

The server acts as a blind relay. Message content is encrypted locally and only readable by the designated recipient.

PFS

Forward Secrecy

Ephemeral Key Exchange

Session keys are refreshed continuously. Even if a future key is compromised, past communications remain secure.

ID

Identity Shield

Digital Signature Verification

Prevents Man-in-the-Middle (MitM) attacks by verifying every node's identity through hardware-bound certificates.

MESH

Node Resilience

Decentralized Trust Model

Eliminates single points of failure. The network maintains integrity even if central nodes are offline or compromised.

IR

Incident Response

Instant Revocation System

Automated protocol to isolate suspicious nodes and re-validate identities within milliseconds of breach detection.

Protocol Operational Flow

Step 01
Hardware Identity Check
TPM Module
The system verifies the node's hardware-bound unique ID to ensure the device hasn't been spoofed or replaced.
Step 02
Ephemeral Key Exchange
Diffie-Hellman (ECDH)
Nodes exchange public parameters to derive a temporary session key that exists only in memory for the duration of the talk.
Step 03
AES-256 GCM Encryption
Encryption Engine
Data is fragmented and encrypted using the AES-256-GCM standard, providing both confidentiality and authenticity.
Active
MitM Attack Detection
Security Monitor
The protocol monitors for timing anomalies and certificate mismatches to detect and block Man-in-the-Middle attempts.
